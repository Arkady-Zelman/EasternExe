### 4. Parallelized ingest pipeline (PR #15, pending)

Replaced sequential per-item chaining with `Promise.all` parallel execution in three route handlers:

- **`extract/route.ts`** — all pending uploads processed concurrently instead of one-per-hop self-chaining
- **`profiles/route.ts`** — all participant profiles generated in parallel instead of sequential `?i=N` indexing
- **`memory/route.ts`** — merged memory + places into a single step running both LLM calls concurrently (they're independent — both just read chunks)

**New chain**: `extract → profiles → memory+places → finalize`
**Old chain**: `extract → extract → ... → profiles?i=0 → profiles?i=1 → ... → memory → places → finalize`

Expected ~2-3x faster ingestion for trips with 3+ participants and multiple uploads.
