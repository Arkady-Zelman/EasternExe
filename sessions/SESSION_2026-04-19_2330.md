## Changes landed (from the cut-off Cursor session, verified in-tree)

### 1. Realtime chat self-heals via polling + immediate POST confirm (`src/hooks/useChatMessages.ts`)

Previously the hook relied entirely on Supabase Realtime, which requires per-table replication toggles in the dashboard. When realtime wasn't enabled on `chat_messages` (or the socket stalled), optimistic messages sat at 50% opacity forever and agent updates never appeared.

Three fixes:
- **3s polling fallback** — `setInterval` fetches `/api/messages?room_id=<id>` every 3 seconds and merges via the existing dedupe (`mergeIn`). Works even if realtime is completely off.
- **Immediate confirm from POST response** — `/api/messages` already returned `{ message: inserted }`; the hook now merges that row straight away instead of waiting for the realtime echo. Optimistic placeholder swaps to the confirmed row within one round-trip.
- **Error-path diagnostics** — `CHANNEL_ERROR` / `TIMED_OUT` subscribe states now warn in console ("relying on polling") so dashboard misconfig is visible.

### 2. More expressive bot animations (`src/app/globals.css`, `src/components/chat/BotAvatar.tsx`)

The `@keyframes` were too subtle to read at a glance (2px float, 5% pulse). Rewritten with bigger translate/rotate/scale:

- `float` — gentle 3px idle bob (unchanged intent, tuned amplitude).
- `pulse-think` — scales 0.95–1.08 with rotation ±5° for a "considering" feel.
- `bounce-speak` — translateY -5px + scale 1.1, then under-shoot to translateY 1px scale 0.95. Snappy.
- `happy-pop` — scale 1.2 with rotation ±8°, then settles at 1.05. Runs **3 iterations** (via `animate-[happy-pop_0.5s_ease-in-out_3]`) so the "done" state is a visible celebration rather than a single flicker.

BotAvatar's `ANIMATION_CLASSES` map updated to match. MessageBubble already wires `thinking_state` → `BotAvatar state` as: `thinking → thinking`, `streaming → speaking`, `done → happy`, else `idle`.
