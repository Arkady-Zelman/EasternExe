## Changes landed

### 1. Subagent receives participant profiles + time-of-day (`src/lib/agent/subagent-research.ts`)

Previously, `runResearchSubagent` received only `description`, `requesterContext`, `destination`, and `tripMemory`. The subagent had zero knowledge of who the people were or what time it was.

Added two new args:
- `profilesJson: string` — full participant profiles with display names, personality, budget style, food preferences, travel style, dislikes, dealbreakers
- `currentTimeOfDay: string` — computed from `new Date()` (morning/afternoon/evening/night)

Both are forwarded into `subagentResearchUser()` prompt builder.

### 2. Main agent computes time-of-day and forwards profiles (`src/lib/agent/main.ts`)

Added `getTimeOfDay()` helper — hour-based heuristic:
- 5–11: morning
- 11–17: afternoon
- 17–21: evening
- 21–5: night

The `spawnResearchSubagent` closure now serializes `profilesForPrompt` into JSON and passes `currentTimeOfDay` alongside the existing args. No new DB queries — profiles were already loaded for the main agent's own context.

### 3. Subagent prompt rewritten with personality, time, and event scanning (`src/lib/prompts/subagent-research.ts`)

**System prompt changes:**
- Instructs the subagent to cross-reference participant personalities, budget styles, food preferences, travel styles, and dealbreakers against every recommendation
- Instructs it to bias searches toward time-appropriate activities based on `currentTimeOfDay`
- Instructs it to always run at least one `web_search` for limited-time events: festivals, pop-ups, seasonal activities, temporary exhibitions, concerts, pop-up markets
- Output format now includes a separate "Happening Now" section for event findings
- Word limit raised from 300 to 350 to accommodate event section

**User prompt template changes:**
- Includes `profilesJson` block with full participant profiles
- Includes `currentTimeOfDay` for temporal context
- Updated "Begin investigating" instruction to explicitly call out event scanning

### 4. Group chat prompt auto-routes suggestions through research (`src/lib/prompts/agent-group.ts`)

Added explicit routing instructions: when the group asks for suggestions, recommendations, activity ideas, itinerary help, or venue/event questions, the agent MUST use `research_activity` instead of answering from general knowledge.

Direct tool use (`search_places`, `query_trip_brain`) reserved for quick factual lookups only.

When calling `research_activity`, the agent is instructed to include a concise `requester_context` summarizing who is asking and what preferences matter.

### 5. Private chat prompt mirrors routing + adds profile-aware context (`src/lib/prompts/agent-private.ts`)

Same routing instructions as the group prompt. Additionally instructs the private assistant to build a rich `requester_context` when calling the research subagent that includes:
- The participant's own budget style, food preferences, travel style, dealbreakers
- Any specific preferences mentioned in the private conversation
- Who else in the group would be involved (if relevant)
